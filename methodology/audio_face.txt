Данный файл описывает методологию генерации в том случае, когда источник - аудио, цель - лицо.

Файлы выбираются следующим образом:
Цель - видео из датасета, идущие подряд. Каждое видео берется один раз.
Источник - аудио. Разберем на примере VoxCeleb2, конкретнее - тестовую выборку из датасета. Структура датасета:
VoxCeleb2_test
|-idxxxxx
 |-#Директории, название которой - индекс видео с YouTube, из которого взяли фрагменты аудио. Обычно на id очень много видео
  |-xxxxx.m4a
  |-xxxxy.m4a #может быть и один аудио файл
......

В качестве источника по порядку сортировки в алфавитном порядке берется: человек (т.е. id), на человека берется 4 разных индекса видео с youtube, c каждого индекса видео берется первый в директории файл .m4a - И данное аудио используется 5 раз.
То есть, каждое аудио берется 5 раз в качестве источника, у каждого человека берется 4 разных аудио из разных интервью. Итого, один человек используется 20 раз для генерации.

Соответственно, первое полученное таким алгоритмом аудио используется в качестве источника для целей 000-004.mp4, первый человек используется в качестве источника для целей 000-019.mp4.

Если генератор не поддерживает видео в качетсве цели, то в качестве цели берется первый кадр из этого видео.

Алгоритм приведен в файле audio_name.py
